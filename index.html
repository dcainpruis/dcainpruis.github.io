project.asteroids — Asteroids

  • Written in Typescript using a functional programming style
  • Utilised Observables for the game loop and handling user input

For a functional programming unit at University, we had to program a version of the game Asteroids. In my version, a user has three lives and a life is lost everytime it collides with an asteroid. My favourite part of this project was implementing the collision algorithm.

First, the algorithm checks if the asteroid and user's ship are close by. Then it calculates vectors for each side of both polygons. Lastly, it checks for any intersections between the ship's vectors and the asteroid's vectors. This allowed more precise collision than a square hit box since the objects are rotating in space. Other notable features of the game are:

  • Destroying a large (red) asteroid splits it into two smaller (orange) asteroids.
  • Asteroids destroyed add to the user score, which increase the level. A higher level increases asteroid density.
  • The user has 3 lives only before game over. Colliding with an asteroid causes the asteroid to explode and a life to be lost.

project.patient_monitor — Patient Monitor

  • User interface created with JavaFX
  • Followed design principles and patterns to create and later extend a class diagram in UML

A desktop application was developed over the course of an entire unit that allowed a health practitioner to sign in with an id and view a list of their patients that can be monitored. The following are a list of features that were implemented:

  • Patients and information regarding their blood pressure and cholesterol are retrieved from a FHIR server.
  • Data from the FHIR is updated on a frequency that is inputted by the user
  • Patients from a list can be selected to monitor their blood pressure, cholesterol, or both in detail.
  • Any patients that have a diastolic blood pressure, systolic blood pressure, or cholesterol above a certain user set threshold are highlighted.
  • Line charts can be displayed to show change in readings over time for multiple patients.

A large part of the software and assignments was to create software using design principles and patterns such that we could later extend the software. The software made use of observers, abstract factories, iterators, and more.

In the later assignment, extending the initial software was easy and required only a few refactors due to how it was initially designed.

project.goodreads_plus — Goodreads+

Goodreads provides a platform for reviewing, rating, and searching for books. When working on this project there was no feature to filter/sort a shelf for a specific genre. This project aims to create a solution to this, which consists of two components:

  1. A web scraper written in python
  2. A web application using django to display the scraped data and sort/filter it

Web Scraper

Currently, only books from the fantasy shelf, https://www.goodreads.com/shelf/show/fantasy, are being scraped as that's the main genre of books that I read. Before book information can be retrieved, a request must be made to login to gain access to more than the first page of the shelf. By analysing the HTML form the required key-value pairs required in the payload can be found and used for logging in.

After logging in, the scraper retrieves the name of each book, the author, the average rating, the number of ratings, the year published, and a thumbnail of the book cover. This information is saved for the application to use and does require updating occasionally as the values do change. Updating the information is as simple as starting the web application and going to the /scraper/ endpoint.

Web application

A simple web application is used to display the data retrieved from the scraper. The web application makes use of Django's templates to generate the HTML page dynamically using the data in the SQLite database. Javascript functionality implemented allows:

  • Exclusion of certain book names/series (i.e. can remove book series I've read)
  • Selection within a range for rating, number of ratings, and year.

The result is as seen in the image below. The UI is simplistic, but it provides the functionality to help me search for books easier and it's found me multiple new books to read.

Future work on the project includes creating a profile remember books that I've read before so they are removed from the list and to also expanding the number of books in the database (currently 1250).

project.book_finder — Book Finder

  • An iOS application that is a work in progress
  • Extends Goodreads+ and adds more features

This is currently being developed as part of an iOS development unit. The aim of the app is the same as Goodreads+, but includes additional features for a user on top of the search functionality and is for iOS instead of a website. The additional features of this application are:

  • Every user can add/remove books to default lists - what they have read or want to read
  • Following the future work of Goodreads+, lists can be used to further filter lists by removing all books present in a selected list
  • Users can rate, and review books or even recommend another book
  • Users can create custom lists, which can be later shared and rated by other users
  • Books will show ratings from outside sources (i.e. Goodreads or Google books) to get as much input from other people as possible
  • The database of books will be initially populated with a selection of books retrieved using the web scraper from the Goodreads+ project
